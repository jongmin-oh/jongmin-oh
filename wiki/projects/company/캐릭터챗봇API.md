# 레플리 메인 — 캐릭터 페르소나 챗봇 서비스 API

**기간**: 2024.01 – 2026.05  
**소속**: [[entities/companies/레플리\|Reppley]]  
**역할**: 설계·구현·운영 전담

## 개요

레플리 메인 제품. 캐릭터 페르소나 챗봇 서비스의 핵심 API. 시장에서 검증된 기능 패턴을 빠르게 흡수해 구현하고, 자사 사용자로 재검증하는 사이클로 운영.

## 주요 기능 5종

| 기능 | 설명 |
|------|------|
| 캐릭터 챗 | 상황·페르소나에 맞는 캐릭터 응답 생성 및 스토리 진행 |
| 속마음 엿보기 | 현재 상황에 맞는 캐릭터의 속마음 생성 |
| 사진 보내기 | 사용자가 보낸 사진을 캐릭터가 이해하고 답변 |
| 페르소나 만들기/다듬기 | 키워드만으로 캐릭터 페르소나 자동 생성 |
| 상황 이미지 | 대화 컨텍스트에 맞는 캐릭터 이미지 노출 |

## 핵심 결정들

### 비용 최적화
일 70만 건 규모에서 비용이 선형 증가하지 않도록:
- **대화 요약 시스템** 도입 — 컨텍스트 축약으로 토큰 절감
- **프롬프트 캐싱** 도입
- 합산 **건당 비용 30% 절감**

### 시장 패턴 흡수
경쟁 제품(캐릭터 챗봇 시장) UX·기능을 분석 → 자사 제품에 맞게 빠르게 구현 → 자사 사용자에게 검증하는 사이클.

## 24/7 운영 & 장애 대응

24시간 멈추면 안 되는 서비스를 일 최고 70만 대화 규모로 무중단 운영. 직접 겪고 잡은 장애들:

| 장애 | 원인 | 재발 방지 |
|------|------|----------|
| gunicorn 워커 타임아웃 | 장시간 요청에 워커가 묶임 | 워커/타임아웃 설정 재조정 |
| LLM 프로바이더 장애 | 모델사 장애가 서비스 중단으로 직결 | 폴백·재시도 대응책 마련 |
| 메모리 폭증(OOM) | 로그 수집 시 클라이언트 객체를 매 요청 생성 | 객체 재사용(싱글턴화) |
| ECS 내부 통신 타임아웃 | 기본 타임아웃 15초를 인지 못함 | 설정 파악·조정 |

> "빠르게 만드는 것"과 "안정적으로 굴리는 것"을 한 사람이 끝까지 책임진 경험. 서버리스/매니지드 우선 설계(운영 부담 최소화)와 대규모 프로덕션 장애 대응 경험이 공존한다.
