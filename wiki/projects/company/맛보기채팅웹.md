# 맛보기 채팅 웹페이지

**소속**: [[entities/companies/레플리\|Reppley]]  
**URL**: https://chat-preview.reppley.com

## 문제

**비오가닉 유저의 앱 가입 전환율 저하.**
- 휴대폰 번호 가입 거부감
- 앱 설치 마찰
→ 가입 전에 제품 가치를 먼저 체험시키는 구조가 필요.

## 해결 방향

기존 가입 UX 개선 대신 **"가입 전 체험 별도 진입점"** 신설로 우회.

앱 서비스지만 웹에서 인기 캐릭터와 맛보기 채팅이 가능한 별도 진입점 → 가입 마찰 우회.

## 주요 기능

| 기능 | 설명 |
|------|------|
| 웹 맛보기 채팅 | 회원가입·앱 설치 없이 인기 캐릭터와 즉시 채팅 체험 |
| 외부 채널 연동 | 틱톡 홍보 영상 → 웹 맛보기로 직접 연결 |
| 공유 링크 바이럴 루프 | 앱 내 공유 → 웹 링크 생성 → 수신자 웹 체험 → 앱 다운로드 |

## 특이사항 — React 미경험에서 구현

React를 이 프로젝트 전에 사용해 본 적 없었음. **AI 코딩 도구(바이브 코딩)** 로 웹페이지 직접 구현·배포.

→ [[skills/working-style\|"읽고 넘기지 않고 바로 적용"]] 원칙의 실례. 미경험 영역을 도구를 활용해 직접 뚫음.

## 협업 — 디자이너와 Figma로 소통하며 개발

혼자 화면을 임의로 짠 것이 아니라 **디자이너와 Figma 시안을 기준으로 소통하며 개발**했다. 디자이너가 Figma로 UI/UX 시안을 잡으면, 그 시안을 기준으로 컴포넌트·레이아웃·인터랙션을 구현하고 디테일을 맞춰가는 협업 흐름.

→ 단순 풀스택 구현을 넘어 **디자인 핸드오프를 읽고 시안대로 프론트를 구현하는 협업 역량**의 실례. 디자이너-개발 사이를 직접 메우며 화면을 완성한 경험.

## 스택

`React` · `AWS S3 + CloudFront + Route 53` · `AI 코딩 도구` · `Figma (디자인 핸드오프)`

## 인프라 패턴

정적 프론트 → S3 + CloudFront로 서버리스 호스팅. [[skills/working-style\|서버리스 우선]] 철학 일관 적용.
